The project

The idea was simple: a model should not only return a class label. It should show why the result was produced.

The app takes a cervical cell image, runs classification, visualizes the influential region, and gives a plain-language explanation for non-technical users. The goal is decision support, not replacement of clinical judgement.

System pieces

  • Image classifier for cervical cytology cell classes.
  • Grad-CAM style visual explanation for model attention.
  • Assistant layer for natural-language result explanation.
  • API backend for serving prediction and explanation outputs.
  • Experiment tracking mindset for training runs and metrics.

What it shows

Trust needs an interface

In healthcare AI, the output has to be inspectable. This project connects prediction, visual evidence, and explanation in one user flow.

Engineering value

It demonstrates the full loop: model training, inference, explainability, backend serving, and a front-end experience that a non-technical user can understand.

Portfolio angle. This is useful as a compact example of explainable AI and product thinking around medical ML systems.
